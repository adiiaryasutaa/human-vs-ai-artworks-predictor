# Human vs AI Art

Django web app that estimates whether a piece of **artwork** — a painting,
drawing, or digital piece — was made by a human or generated by AI, using a
trained EfficientNetV2 model. It's built for art, not photos or screenshots.
UI is server-rendered with Flowbite/Tailwind (CDN), no build step.

## Pages

- `/` — landing page: what it is, how it works, an honest "about the model"
  section, and an FAQ.
- `/detect/` — the detector: submit an artwork and read the verdict.

## Features

- Submit by **file upload**, **direct URL**, **clipboard paste**, or **camera**.
- Live preview of the selected artwork *before* you analyze.
- **Async analysis** — the verdict (and the analyzed image) render inline without
  a full page reload; the model is warmed in the background when `/detect/` opens.
- Client-side validation (PNG/JPG/WEBP, ≤ 15 MB) mirroring the server caps.
- Optional "try one" examples — drop images into
  `detector/static/detector/examples/` and the row turns on automatically.
- Verdict shown as a **Human ↔ AI confidence meter**.

## Setup

Requires Python 3.12 (the model needs Keras ≥ 3.11, which dropped 3.9/3.10).

```bash
python3.12 -m venv .venv
.venv/bin/pip install -r requirements.txt
```

### Model weights

The `.keras` model files are **not in the repo** (too large — see `.gitignore`).
For local dev, place a trained model in `models/` (default path:
`models/best_model4.keras`, set by `ML_MODEL_PATH` in `config/settings.py`).

## Deploy (Hugging Face Spaces, Docker)

The repo ships a `Dockerfile` and HF Space config (YAML header in this README).
Weights are pulled from a Hugging Face model repo at runtime, not baked into the
image.

1. Upload the model to a public HF model repo:
   ```bash
   huggingface-cli upload <user>/<repo> models/best_model4.keras best_model4.keras --repo-type model
   ```
2. Create a Space (SDK: **Docker**) and push this repo to it.
3. Set Space **variables**: `ML_MODEL_REPO=<user>/<repo>`, `SECRET_KEY=<random>`,
   `DEBUG=False`. (Private model repo → also add `HF_TOKEN` as a secret.)

Notes: serve over the **direct `*.hf.space` URL** for camera capture to work
(the embedded Space iframe blocks `getUserMedia`). Static files are served by
WhiteNoise; `collectstatic` runs at image build.

## Run

```bash
.venv/bin/python manage.py runserver
```

Open http://127.0.0.1:8000. No database is used, so there is nothing to migrate.

## Project layout

- `config/settings.py` — Django settings (no DB, no auth, no sessions) + `ML_MODEL_PATH`
- `detector/ml.py` — model singleton (`get_model`), background warmup
  (`warm_model_async`), and inference (`predict_image`)
- `detector/urlfetch.py` — fetch images from a URL with SSRF guards
- `detector/views.py` — `landing` + `detect` views (AJAX returns the result partial)
- `detector/templates/detector/` — `base.html`, `landing.html`, `index.html`,
  `_result.html`

## Notes

- The model has rescaling built in — the app feeds raw `[0, 255]` pixels.
- Input size is read from the model at startup, so retraining at a different
  resolution needs no app changes.
- Verdict mapping is controlled by `HUMAN_IS_ONE` in `detector/ml.py`. If verdicts
  come out inverted on known artwork (human called AI and vice versa), flip it.
- Predictions between `UNCERTAIN_LOW` (0.4) and `UNCERTAIN_HIGH` (0.6) are reported
  as "Not sure". `USE_FLIP_TTA` averages each artwork with its mirror.
- Results are estimates, not proof. Unfamiliar generators or heavily re-compressed
  art can fool the model.
